Como criar um repositório usando github

Vídeo: https://youtu.be/-l4Aa8wef8s

GIT permite controlar versões do código. Uma linha do tempo da criação do projeto.

COMMIT é o registro de uma alteração na linha do tempo do projeto.

Ordem de eventos:

git init

Inicia o projeto (importante verificar se no compilador o seu github já está conectado).

git add .
git commit -m "mensagem do primeiro commit"

Adiciona todos os arquivos modificados (o ponto é para todos, mas pode ser usado apenas com o nome de um arquivo) e realiza o commit dos arquivos para o seu repositório.

git branch -M main

Depois de dar o commit, é importante criar a branch MAIN do projeto.

git status

Acompanha o status do repositório (alerta mudanças não commitadas).

git log

Visualiza o histórico de mudanças do código.

git diff

Mostra a diferença do código atual para o último commit realizado.

git restore

Recupera a versão do último commit para o código atual.

git branch

Mostra qual a branch em que o usuário está. A branch é importante porque cria ramificações de versões do código, e quando tem muitas pessoas usando, cada uma pode ter sua branch para realizar as mudanças.

git checkout -b nome-da-branch

Cria uma nova branch a partir da branch atual, levando todo o código salvo até o momento para a nova ramificação.

git checkout master

Sendo master o nome da branch principal, o usuário volta para ela usando esse comando. Trocando o nome da branch, o usuário é levado para a branch que selecionou.

git checkout nome-da-branch
git merge outra-branch

O merge vai adicionar as alterações de outra branch na branch onde o usuário se encontra.

Repositório Remoto

Agora para criar um repositório remoto e salvar essas alterações em nuvem, é necessário criar o repositório primeiro dentro do github.

Uma vez que o repositório foi criado, é preciso pegar o link do repositório para vincular o código a ele, sendo a ordem de eventos:

git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/LouzadaCaetano/UsandoGit.git
git push -u origin main

Como os outros passos já foram realizados (exceto o README.md), só precisa adicionar o remote origin com o link do repositório e dar o PUSH para salvar o projeto.

Nesse projeto, o README foi adicionado no GitHub, e não na máquina, e para puxar alterações para a máquina, usamos:

git pull origin nome-da-branch

Esses são os comandos importantes do GIT para um projeto, seja ele individual ou em equipe.
